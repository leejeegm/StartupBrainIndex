## 1. 필요한 환경 (HW/SW) — 미리 확인

아래만 갖추면 됩니다. **모두 갖춰졌는지 체크**해 보세요.

### 1-1. 하드웨어(HW)

| 항목 | 조건 | 확인 |
|------|------|------|
| PC | Windows 10 이상 (또는 Mac; 여기서는 Windows 기준으로 설명) | ☐ |
| 인터넷 | 웹 브라우저로 사이트 열 수 있는 연결 | ☐ |
| (선택) 모바일 테스트 | 같은 Wi‑Fi에 붙일 스마트폰/태블릿 | ☐ |

### 1-2. 소프트웨어(SW)

| 항목 | 어떻게 확인·설치 | 확인 |
|------|------------------|------|
| Python 3 | 명령 프롬프트에서 `python --version` 입력 → `3.8` 이상 나오면 OK. 없으면 [python.org](https://www.python.org/downloads/)에서 설치 시 **"Add Python to PATH"** 반드시 체크 | ☐ |
| 프로젝트 폴더 | SBI 프로젝트가 압축 해제된 폴더 경로 (예: `C:\vibeTest\project2_StartupBrainIndex`). 이 경로를 **메모장에 적어 두기** | ☐ |

**검증 1 — Python 확인**  
1. 키보드 `Win + R` 누르기 → `cmd` 입력 → Enter  
2. 창에 `python --version` 입력 후 Enter  
3. **성공**: `Python 3.x.x` 같은 줄이 나옴  
4. **실패**: "python을 찾을 수 없습니다" → Python 설치 시 "Add Python to PATH" 체크 후 재부팅하고 다시 시도

## 2. 가장 쉬운 방법 — 로컬에서 먼저 실행 (무료, 에러 최소)

**순서**: 폴더 열기 → 배치 파일 한 번 실행 → 브라우저로 접속 확인.  
이 과정만 통과하면 "시스템 환경 셋팅"은 끝난 것입니다.

### 2-1. 한 번만 할 일: 라이브러리 설치

1. **명령 프롬프트** 열기 (Win + R → `cmd` → Enter)
2. 아래 한 줄을 **그대로 복사**해서 붙여넣기 후 Enter.  
   (경로는 본인 프로젝트 폴더로 바꾸세요.)

```
cd /d C:\vibeTest\project2_StartupBrainIndex
```

3. 그 다음 한 줄 실행:

```
pip install -r requirements.txt
```

4. **성공**: 마지막에 `Successfully installed ...` 비슷한 문구  
5. **실패**: `pip을 찾을 수 없습니다` → Python 설치 시 "Add Python to PATH"가 안 된 것. Python 재설치 후 위 2~3 다시

**검증 2**  
같은 창에서 `pip list` 입력 후 Enter → `fastapi`, `uvicorn` 등이 목록에 보이면 OK.

### 2-2. 서버 켜기 (매번 사용할 때)

1. **파일 탐색기**에서 프로젝트 폴더 열기 (예: `C:\vibeTest\project2_StartupBrainIndex`)
2. **`run_server.bat`** 파일을 **더블클릭**
3. **검은 창(콘솔)** 이 뜨고, 대략 아래처럼 나오면 **성공**:

```
[SBI] 서버를 시작합니다.
[SBI] PC 접속: http://127.0.0.1:8000/
[SBI] 모바일 접속: http://192.168.x.x:8000/  (같은 Wi-Fi에서만)
*** 이 창을 닫지 마세요. 닫으면 서버가 종료됩니다. ***
```

4. **이 검은 창을 닫지 마세요.** 최소화만 해 두기.

**검증 3 — PC에서 접속**  
1. 웹 브라우저(Chrome 등) 열기  
2. 주소창에 `http://127.0.0.1:8000` 입력 후 Enter  
3. **성공**: SBI 첫 화면(시작하기 버튼 등)이 보임  
4. **실패**: 아래 "4. 접속 오류 시 대응" 참고

**검증 4 — 기본 기능**  
- "시작하기" → 로그인 화면 → 데모 계정 `user@test.com` / `pass1234` 로 로그인 → 대시보드까지 들어가 보기.  
  여기까지 되면 **로컬 실행·환경 셋팅은 정상**입니다.

## 4. 서버 접속 오류·단절 시 점검 및 대응

에러가 나도 **아래 순서대로만** 확인하면 원인을 좁힐 수 있습니다.

### 4-1. "사이트에 연결할 수 없음" / "연결이 거부되었습니다"

| 확인할 것 | 조치 |
|-----------|------|
| **검은 창(서버)이 떠 있는가?** | 닫혀 있으면 **`run_server.bat`** 다시 더블클릭해서 서버부터 실행 |
| **주소 오타** | `http://127.0.0.1:8000` (띄어쓰기 없이, 숫자 8000) |
| **포트가 8000이 아닌 경우** | 검은 창에 나온 **실제 주소**(예: `http://127.0.0.1:8001`)로 접속 |
| **방화벽** | Windows 방화벽에서 "Python" 또는 포트 8000 허용 여부 확인. 처음 실행 시 "액세스 허용" 나오면 허용 선택 |

### 4-2. "이 페이지가 작동하지 않습니다" / 502, 503

- **로컬**: 서버(검은 창)가 꺼졌을 가능성이 큼. `run_server.bat` 다시 실행 후 새로고침.
- **Render 등 클라우드**: 서비스가 슬립 상태일 수 있음. 1~2분 기다렸다가 다시 접속. (무료 플랜은 일정 시간 미사용 시 슬립)

### 4-3. 서버가 갑자기 꺼짐 (단절)

| 원인 | 대응 |
|------|------|
| **검은 창을 닫음** | 서버는 그 창이 살아 있어야 동작함. **`run_server.bat`** 다시 더블클릭해서 재실행 |
| **PC 절전/슬립** | 서버도 함께 멈춤. 절전 설정 완화 또는 "절전 안 함"으로 변경 (설정 → 전원 및 절전) |
| **오류로 종료** | 검은 창에 빨간 글씨/에러 메시지가 남아 있으면, 그 문구를 복사해 두고 필요 시 지원처에 전달. 일단 **`run_server.bat`** 다시 실행 후 접속 재시도 |

### 4-4. 모바일에서 "연결할 수 없음"

| 확인 | 조치 |
|------|------|
| **같은 Wi‑Fi인가?** | PC와 휴대폰이 **같은 공유기(Wi‑Fi)** 에 연결되어 있어야 함 |
| **주소** | PC가 아닌 **모바일 접속 주소** 사용. 검은 창에 나온 `http://192.168.x.x:8000` (숫자는 PC마다 다름) |
| **PC IP 확인** | 모르면 PC에서 명령 프롬프트 열고 `ipconfig` 입력 → "IPv4 주소"가 192.168.x.x 형태 → `http://그주소:8000` 으로 접속 |

### 4-5. DB 관련 오류 ("데이터베이스에 연결할 수 없습니다")

- **로컬에서 SQLite 사용**: 별도 DB 설치 없이 실행만 하면 됨. 위 2-1 단계만 제대로 했는지 확인.
- **Render + MySQL(닷홈 등)**: Render 대시보드의 **Environment**에 `DB_ENGINE`, `DB_HOST`, `DB_USER`, `DB_PASSWORD`, `DB_NAME`이 **정확히** 들어갔는지 확인. 닷홈은 "원격 접속" 가능 여부 확인 필요 → `닷홈_연동_재배포시_데이터유지_가이드.md` 참고.

## 5. 주의사항 정리 (접속·단절 방지)

1. **서버 창(검은 창)을 닫지 않기**  
   닫는 순간 서버 종료 → 접속 불가. 최소화만 하기.

2. **PC 절전 끄기**  
   장시간 테스트 시 설정 → 시스템 → 전원 및 절전에서 "PC를 슬립 모드로 전환" → **안 함** 권장.

3. **재부팅/종료 후**  
   PC를 다시 켠 뒤에는 **매번 `run_server.bat`** 다시 실행해야 접속 가능.

4. **데이터를 잃지 않게 (시범운영)**  
   재배포해도 회원·설문 데이터를 유지하려면 **DB를 앱과 분리**해야 함.  
   → `닷홈_연동_재배포시_데이터유지_가이드.md`, `GitHub_무료_시범운영_가이드.md` 참고.

## 7. 요약 — "따라하기 + 검증" 한 줄 요약

1. **Python 설치**(PATH 포함) → `python --version` 으로 검증  
2. **프로젝트 폴더**에서 `pip install -r requirements.txt` → `pip list` 로 검증  
3. **`run_server.bat`** 더블클릭 → 검은 창에 접속 주소 나오는지 검증  
4. **브라우저**에서 `http://127.0.0.1:8000` 접속 → 첫 화면·로그인까지 검증  
5. **접속 오류/단절** 시: 서버 창 유지 여부 → 주소·Wi‑Fi·방화벽 → 서버 재실행 순으로 확인  

이 순서만 지키면 **HW/SW 환경 셋팅과 서버 접속 오류·단절 대응**을 비개발자도 따라 하며 검증할 수 있습니다.
